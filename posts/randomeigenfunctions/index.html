Sum of squares

Determining if an integer nnn satisfies this property, to be the sum of two squares, is an old problem with a long history - it dates back to Diophante. Fermat gave the first characterization of such prime numbers. The general theorem is as follows.

An integer n⩾1n \geqslant 1n⩾1 can be written as the sum of two squares if and only if, in its prime decomposition, the primes ppp with of the form p=4k+3p = 4k + 3p=4k+3 have an odd power. If it is the case, then the number of ways r(n)r(n)r(n) to write nnn as a sum of two squares is given by

r(n)=4(d1(n)−d3(n))r(n) = 4(d_1(n) - d_3(n))r(n)=4(d1​(n)−d3​(n))

where dk(n)d_k(n)dk​(n) is the number of divisors of nnn, not necessarily primes, which are congruent to kkk modulo 444.

For example, 10=2×510 = 2 \times 510=2×5; it has exactly four divisors 1,2,5,101,2,5,101,2,5,10, two of which are congruent to 1 modulo 4, two others to 2. Consequently, d1(10)=2d_1(10) = 2d1​(10)=2 and d3(10)=0d_3(10) = 0d3​(10)=0 and r(10)=8r(10) = 8r(10)=8. These "sum-of-squares-representations" of 10 are

12+3232+12(−1)2+32(−3)2+1212+(−3)232+(−1)2(−1)2+(−3)2(−3)2+(−1)2\begin{aligned} & 1^2 + 3^2 &&&&&&& 3^2 + 1^2\\ & (-1)^2 + 3^2 &&&&&&& (-3)^2 + 1^2\\ & 1^2 + (-3)^2 &&&&&&& 3^2 + (-1)^2\\ & (-1)^2 + (-3)^2 &&&&&&& (-3)^2 + (-1)^2\\ \end{aligned}​12+32(−1)2+3212+(−3)2(−1)2+(−3)2​​​​​​​32+12(−3)2+1232+(−1)2(−3)2+(−1)2​

Here is a little function for generating all these representations (it is by no means optimized in any way):

function SOSrep(n::Int)
    """ Returns the list of Sum-Of-Squares representations.
    Returns an empty list if there are none. """
    square_reps = []
    for i in 1:sqrt(n)-1
        if isinteger(sqrt(n - i^2))
            j = sqrt(n - i^2)
            append!(square_reps,  [[i,j], [i,-j], [j,i], [j,-i]])
        end
    end
    if isinteger(sqrt(n))
        j = sqrt(n)
        append!(square_reps, [[0,j], [0, -j], [j, 0], [-j, 0]])
    end
    return square_reps
end

Random arithmetic waves

Let us go back to the Laplace equation (1). Let nnn be an integer that can be written as a sum of squares and let λn\lambda_nλn​ be the associated eigenvalue. Then, the subspace En\mathscr{E}_nEn​ of solutions of (1) has dimensions r(n)r(n)r(n), the number of ways to write nnn as a sum of squares discussed above. Let S\mathscr{S}S be the set of couples x=(x1,x2)x=(x_1,x_2)x=(x1​,x2​) with n=x12+x22n= x_1^2 + x_2^2n=x12​+x22​; then, a basis of En\mathscr{E}_nEn​ is given by the elementary functions

ex: (z1,z2)∈T↦exp⁡{2iπ(x1z1+x2z2)} e_{x} : (z_1, z_2) \in \mathbb{T} \mapsto \exp\{ 2i\pi (x_1 z_1 + x_2 z_2) \} ex​: (z1​,z2​)∈T↦exp{2iπ(x1​z1​+x2​z2​)}

and the solutions of (1) are all the linear combinations of these r(n)r(n)r(n) functions, namely the functions of the form

∑x∈Sαxex \sum_{x \in \mathscr{S}} \alpha_x e_x x∈S∑​αx​ex​

where αx\alpha_xαx​ are complex numbers. What we call random arithmetic waves are random elements of En\mathscr{E}_nEn​ obtained by choosing the αx\alpha_xαx​ to be iid standard complex Gaussian variables[1]. The resulting random function fnf_nfn​ is a random Gaussian field; it can be viewed as a typical element of En\mathscr{E}_nEn​, and there is a vast litterature on its behaviour. Usually, it is more natural to study real Gaussian fields, and for this we have to ensure that fnf_nfn​ is real by enforcing some minor constraints on the αx\alpha_xαx​: we simply ask that α−x=αx‾\alpha_{-x} = \overline{\alpha_x}α−x​=αx​​ (complex conjugate).

We generate a random arithmetic wave with the following function:

function generate_RAW(n::Int)
    Λ = SOSrep(n)
    if isempty(Λ)
        return nothing
    else
        E = [(x,y) -> exp(2im*pi*(λ[1]*x + λ[2]*y)) for λ in Λ] #exponentials
        X = [randn() + 1im*randn() for λ in Λ]
        f(x,y) = sum(real(X.*[e(x,y) for e in E])) / sqrt(length(Λ))
        return f
    end
end

Let us generate some arithmetic wave; most numbers have very few sum-of-squares representations (0,4 or 8), but 85=5×1785 = 5 \times 1785=5×17 has 16 of them. 

f = generate_RAW(85)

Also, 71825 has 68 square representations, and 801125 has 124, which is a lot.

A torus can be parametrized with two coordinates (θ,φ)∈[0,2π)2(\theta, \varphi) \in [0,2\pi)^2(θ,φ)∈[0,2π)2 by the following formulas:

X(θ, φ) = (2 + cos(θ)) * cos(φ)
Y(θ, φ) = (2 + cos(θ)) * sin(φ)
Z(θ, φ) = sin(θ)
